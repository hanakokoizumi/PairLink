# Coturn (TURN)

PairLink starts coturn automatically via Docker Compose. Configuration is driven by environment variables in `.env` — **do not edit files here manually**.

Run once before first deploy:

```bash
make setup
```

This generates `TURN_PASSWORD` and wires credentials into both PairLink (`/api/config` → `rtcConfig`) and coturn.

## Variables

| Variable | Description |
|----------|-------------|
| `TURN_ENABLED` | `true` to start coturn and expose TURN in API config |
| `TURN_HOST` | Hostname clients use (default: host from `PUBLIC_URL`) |
| `TURN_PORT` | Listening port (default `3478`) |
| `TURN_USER` / `TURN_PASSWORD` | Long-term credentials (password auto-generated by `make setup`) |
| `TURN_REALM` | coturn realm (default `pairlink`) |
| `TURN_EXTERNAL_IP` | Public IP for NAT traversal (optional) |

Set `TURN_ENABLED=false` to disable the bundled TURN server.
